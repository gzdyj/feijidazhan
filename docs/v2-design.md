## 一、第一版现状分析

### 1.1 项目结构
- `index.html`：自包含运行版（CSS + JS 全部内嵌，约 1500 行），实际运行以此为准
- `js/`：拆分开发版（`main.js`/`audio.js`/`game.js`/`ui.js`/`all.js`），仅供参考
- `css/`：样式文件

### 1.2 第一版道具系统现状
**掉落道具**（`powerupTypes`，15% 概率掉落，拾取即时生效）：
| 道具 | 图标 | 效果 | 持续 |
|------|------|------|------|
| 双倍火力 | 🔥 | 武器等级+1 | 5秒 |
| 护盾 | 🛡️ | 无敌护盾 | 5秒 |
| 生命 | ❤️ | 生命+1 | 即时 |
| 急速 | ⚡ | 移速×1.5 | 3秒 |
| 金币 | 🪙 | +10金币 | 即时 |

**必杀技**：💣 炸弹（空格键/按钮），清屏秒杀，900帧(15秒)冷却

**商店道具**：额外生命/初始护盾/开局加速（仅扣金币，无实际生效逻辑）

### 1.3 第一版已知 BUG（见 `v2-pitfalls.md`）
1. `useAbility` 未在 `Game` return 中导出 → 必杀技按钮报错
2. 商店道具购买后无实际效果
3. `collectPowerup` speed 恢复未乘 `levelBonuses.speed`

## 二、第二版设计目标

### 2.1 核心方向
1. **主动技能道具**：玩家携带并主动释放
2. **扩充掉落道具种类**：在现有掉落系统上新增更多种类

### 2.2 设计原则
- 不破坏第一版核心玩法，增量扩展
- 道具效果差异化，避免同质化
- 槽位有限（3个），增加策略性
- 视觉反馈清晰，效果可感知

## 三、道具体系设计

### 3.1 新增主动技能道具（携带式，主动释放）

| 道具 | 图标 | ID | 效果 | 冷却 |
|------|------|----|------|------|
| EMP脉冲 | 💫 | emp | 瘫痪全场敌机3秒（停止移动和射击） | 无（消耗品） |
| 时间减速 | ⏱️ | slowmo | 全场敌机减速50%，持续5秒 | 无（消耗品） |
| 磁铁 | 🧲 | magnet | 吸引全场掉落物到玩家，持续8秒 | 无（消耗品） |
| 核弹 | ☢️ | nuke | 对全场敌机造成大量伤害（普通秒杀，BOSS减半血） | 无（消耗品） |
| 分身 | 👥 | clone | 生成2个分身协助射击，持续8秒 | 无（消耗品） |

### 3.2 新增掉落道具（即时/持续效果）

| 道具 | 图标 | ID | 效果 | 持续 |
|------|------|----|------|------|
| 穿透弹 | 🎯 | pierce | 子弹穿透敌机 | 6秒 |
| 双倍分数 | ✨ | double | 得分翻倍 | 8秒 |
| 炸弹补给 | 💣 | recharge | 重置必杀技冷却 | 即时 |

### 3.3 道具分类机制
- `category: 'instant'`：即时生效（生命、金币、炸弹补给）
- `category: 'buff'`：持续增益（火力、护盾、急速、穿透弹、双倍分数）
- `category: 'active'`：主动技能，拾取后进入槽位（EMP、时间减速、磁铁、核弹、分身）

### 3.4 槽位系统
- 玩家最多 3 个道具槽位
- 拾取 active 道具时：槽位未满则入槽，已满则自动使用最旧槽位道具后入槽
- 使用方式：点击槽位图标 或 按数字键 1/2/3
- 槽位道具在 HUD 底部显示

## 四、数据结构设计

### 4.1 道具定义结构
```js
{
  id: 'emp',
  name: 'EMP脉冲',
  icon: '💫',
  category: 'active',   // instant | buff | active
  effect: 'emp',
  dur: 0,               // buff 持续帧数，instant/active 为 0
  desc: '瘫痪全场敌机3秒',
  color: '#00d4ff'
}
```

### 4.2 槽位数据
```js
inventory = [null, null, null];  // 3个槽位，存道具定义对象
```

### 4.3 持续效果状态
```js
activeEffects = {
  emp: 0,           // 剩余帧数
  slowmo: 0,
  magnet: 0,
  pierce: 0,
  double: 0,
  clone: 0
};
```
